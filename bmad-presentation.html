The Product

What is 3D Print Flow?

A local-first web app for 3D printing color-first workflow management. No backend, no cloud, no accounts — just IndexedDB persistence and JSON export.

“What can I print with the spool loaded right now?” instead of “What figure should I print next?”
🎨
Color-First Planning
Ranked list of filament colors sorted by queue demand. Load a spool, see everything you can print.
⚪
Per-Figure Color Chips
Tap to mark a color as printed. Progress fills as each chip is toggled. The signature interaction.
📡
Local-First Architecture
No backend, no cloud, no accounts. All data stays in IndexedDB with JSON export for backup.
⚡
Zero Loading Spinners
In-memory state with write-behind persistence. Every operation is instant from the user's perspective.

The Polish

The UI/UX Overhaul

A comprehensive visual refresh on a dedicated feature branch, driven by the interplay of multiple design skills.

21 files modified feat/ui-design-overhaul branch Multiple review passes
OKLCH Color System
Migrated to warm-tinted neutrals using OKLCH (hue 50–65 for warm range). Not cool gray, not clinical. Let the user's filament colors shine while the UI chrome recedes with workshop warmth.
Enhanced Accessibility
WCAG AA target with proper contrast ratios, focus indicators, semantic HTML, and reduced-motion support. Every interaction is usable with keyboard and screen readers.
Typography & Animations
Purposeful micro-interactions: fade-in-up (400ms), slide-in (300ms), completion-pulse (500ms). All respect prefers-reduced-motion. Inter Variable for clean, readable text.
Design Skill Interplay
/frontend-design generated the code, /audit validated quality scores, .impeccable.md provided persistent design context, and /vercel-react-best-practices ensured optimal React patterns.

The Learnings

Key Takeaways

What building an entire product with BMAD and AI skills revealed about the future of software development.

01
BMAD provides structure that AI alone lacks
Without systematic phases, AI produces scattered, inconsistent output. BMAD channels AI capability through proven software engineering methodology — brainstorming, requirements, architecture, implementation — so nothing falls through the cracks.
02
Rigorous planning produces refined output
47 ideas distilled through 12 PRD validation steps into a 5/5 quality spec. The implementation inherits this quality. Vague thinking upstream means vague artifacts downstream — BMAD enforces rigor at every stage.
03
Specialized agents bring domain expertise
9 agent personas each bring focused perspective. Mary the Analyst thinks differently than Winston the Architect. This specialization mirrors real team dynamics and produces richer, more considered output than a single generalist AI.
04
Method + Skills + MCP = production-grade results
BMAD for structure, AI skills (/frontend-design, /audit, /vercel-react-best-practices) for execution quality, MCP servers (shadcn, GitHub) for seamless automation. The combination exceeds what any single tool achieves alone.
